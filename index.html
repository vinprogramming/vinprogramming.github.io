If there is print function in init, then it will print the sentence when the object is called or a new instance is created. No need to give print fn in main code again.

The number of parameters we pass into instance must be equal to that in init constructor

Without self keyword and dot, we only create a local variable and not an attribute

Behaviors or methods must always have self as their first parameter

If you print the object or instance created in the main code, an address value will be printed. To avoid this, str constructor should be used

You can call one method from another method

If self.attribute_name is not used in init constructor, it becomes a local variable and also it cannot be accessed in the main code.

Summary of Reference Variable:

  1. Reference variables hold the objects
  1. We can create objects without reference variable as well
  1. An object can have multiple reference variables
  1. Assigning a new reference variable to an existing object does not create a new object

A local variable is not attached to any object

OOPs basics summary

  • OOP is a style of programming which allows us to club data and behavior together.
  • This is more suited for coding real life scenarios.
  • Objects are real world entities
  • Class is just a classification. It is just a concept.
  • Class is a description of attributes and behavior that objects of that classification should possess.
  • Attributes are created in a special function called __init__ and behaviors are created using functions called methods.
  • Objects can be created using ClassName() or using object literals for some of the built in classes
  • Attributes are created using reference_variable.attribute_name = value syntax.
  • Behavior is created by defining a function inside the class having a special parameter called sel
